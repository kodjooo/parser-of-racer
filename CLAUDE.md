# CLAUDE.md — инструкции для AI-агента (Codex и др.)

Основные правила  описаны в 'docs/arch-rules.md'
Бизнес требования к сервису описаны в 'docs/requirements.md'

При изменении проекта обязательно нужно вносить описание архитектуры в документ 'docs/architecture.md' использовать его в работе. Также нужно актуализировать '.env.example', '.env' файлы и тесты в папке 'tests'

После реализации каждого этапа плана из файла 'docs/plan.md' нужно обязательно отмечать его там (писать 'выполнено'), как выполненный, но не удалять

В .env файл пиши комментарии, какой доступ и где нужно найти.

В GITHUB не нужно отправлять .env файл 

По завершению работ пушь изменения в гитхаб. Репозиторий проекта: git@github.com:kodjooo/parser-of-racer.git (remote `origin`). Инициируй его, если это еще не сделано. Информацию по разворачиванию проекта на удаленном сервере нужно писать в readme.md.

Если тебе нужна какая-то документация, сначала попробуй найти ее в MCP Context7 и если ее там нет - попробуй найти ее в интернете.

Если тебе нужно протестировать что-то во фронтенде - сделай это через MCP Playwright.



Для работы с удаленным сервером используй SSH-доступ пользователя `developer`.

Когда пользователь просит что-то посмотреть или проверить именно на сервере (например, логи, статус docker compose, файлы проекта, конфиги, запущенные сервисы), сначала подключайся по SSH под `developer` на сервер `159.69.188.45` в директорию `/usr/local/sheets-to-wp-racefinder/parser-of-racer/` по SSH ключу `id_rsa` и выполняй диагностику там.

Если запрос касается рабочей БД на сервере, используй либо SSH-доступ на сервер.
