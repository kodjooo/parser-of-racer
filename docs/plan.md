Этап 9. Улучшенное сопоставление трасс (дедупликация)
Внедряется по требованию заказчика, чтобы в Missing races не попадали трассы,
которые по сути уже есть в RACES, но под другим URL.
- Подэтап 9.1 (D + A) — выполнено
  - app/integrations/matching.py: KnownIndex (exact + parent/child + языковой префикс),
    is_service_page (категория D).
  - Категория C (новый год = новая трасса) соблюдается автоматически.
  - Интеграция в main.py, новые параметры .env, тесты tests/test_matching.py.
- Подэтап 9.2 (B — кросс-платформенно по slug/ID) — выполнено
  - KnownIndex.by_slug + уровень B в match: совпадение одного события на разных
    платформах/поддоменах по slug события (с годом).
  - Защиты: минимальная длина slug, наличие букв, стоп-лист общих слов; каждое
    B-совпадение логируется для аудита; включается флагом CROSS_PLATFORM_MATCH.
  - Тесты категории B в tests/test_matching.py.
